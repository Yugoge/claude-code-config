Root Cause
Symptom: Slash commands (e.g., /dev) not triggered when message includes image attachments.
Root Cause: runClaude.ts:431 constructed the message as  ${refs}\n\n${text}  — attachment @path references were prepended before user text. Claude SDK detects slash commands from the start of the message string, so the @ prefix masked the / command.
Introduced by: Commit f8b208fa (F2 image upload feature).
Fix Applied
File: /root/happy/packages/happy-cli/src/claude/runClaude.ts:431-432
Before:
typescript
const textWithAttachments = refs ? `${refs}\n\n${message.content.text}` : message.content.text;
After:
typescript
const text = message.content.text;
const textWithAttachments = refs ? (text ? `${text}\n\n${refs}` : refs) : text;
- User text now comes first, attachment refs appended after
- Empty text guard: attachment-only messages produce clean @path string without leading newlines
- Comment updated from "prepend" to "append"
