## Edge Cases & Risks

- **Shard scan scope**: detection must be per-TASK_ID, not global — unrelated parallel cycles in other task directories must not trigger serialization for the current task.
- **PARALLEL_AGGREGATE_WRITTEN is not a persistent shell env var**: shell vars set in one Bash call do not persist to the next. Step 0 must record the result to a temp file or capture script stdout that close.md reads in Step 1.
- **NON_WORKER_LABELS parity**: aggregate-dev-report.py without NON_WORKER_LABELS/NON_WORKER_LABEL_RE would classify iteration shards (e.g., `dev-report-TASKID-iter2.json`) as parallel workers, inflating worker count and corrupting validation.
- **task_id top-level field missing from aggregate schema**: `commands/dev.md:729-745` listed `request_id` but omitted `task_id`; `commands/close.md:163` requires `task_id` at the top level. Script must write both.
- **auto_bulk_sha ambiguity in multi-repo setups**: using a single sha is ambiguous. Use `auto_bulk_commits[]` array with `{repo_root, branch, sha}` objects.
- **Retry scope too broad**: retrying all `failed` statuses would retry real git/staging errors. changelog-analyst must emit a `failure_code` field; commit.md retry logic must check failure_code and only retry recoverable grant-failure codes.
- **Grant revocation inline logic**: commit.md cannot use `find /tmp/... -delete` inline (violates no-inline-scripts rule). Must use `scripts/write-commit-grant.py --revoke-existing-for-task` or a new named helper.
- **Fix A+D must land together**: partial application (A without D, or D without A) leaves the pipeline in a worse state than before.
- **Fix B_contract+P5 must land together**: P5 depends on the machine-readable status fields that B_contract defines.
- **Fix C 4-location atomicity**: updating only the constant but not the text references creates doc/code drift. All 4 locations must be in one atomic commit.
