# BA Specification: Changelog-analyst warn-and-skip for provenance-anomaly files in BULK=false mode

**Request ID**: 20260527-133200
**Created**: 2026-05-27T13:32:00Z
**Tier**: SMALL

## Goal

Strengthen the changelog-analyst provenance filter so that in BULK=false mode, any file listed in `dev.files_modified` that does NOT appear in `git diff <baseline_head_sha>` is warned about AND excluded from staging (warn-and-skip), rather than being staged with a warning that it is anomalous. This eliminates the current escape hatch where a dev subagent self-reporting phantom files can still cause those files to be committed.

## Context

Items 1, 2, and 4 of the user requirement are already implemented:
- `commands/dev.md` lines 712-745: orchestrator captures `baseline_head_sha` + `baseline_dirty_snapshot` and passes them to dev dispatch.
- `agents/dev.md` lines 507-522: dev MUST derive `files_modified` from `git diff --name-only <baseline_head_sha>` and report `observed_preexisting[]` separately.
- `agents/qa.md` lines 405-455: QA raises critical FAIL (`dev_provenance_violation`, `files_modified_underreport_violation`) for mismatches.

Item 3 is the genuine gap: `agents/changelog-analyst.md` lines 171-173 classify `provenance_anomaly` paths but then say "Stage them if they appear in the candidate set" — anomalous BULK=false files are staged despite being flagged. The user's requirement is to change this to warn-and-skip (exclude from staging, do not abort the commit).

## Evidence (Contract A)

- **Observed**: dev subagent self-reports `files_modified` which changelog-analyst uses for staging without verifying against git diff. A phantom file in `files_modified` passes into the staged set even though the provenance filter correctly classifies it as anomalous.
- **Measured**: `agents/changelog-analyst.md` line 173: "Stage them if they appear in the candidate set (for BULK=true, staging authority comes from git status; for BULK=false, staging authority comes from the whitelist), but do not use their paths to determine commit type or scope." — anomalous files are explicitly permitted to be staged.
- **Expected**: In BULK=false mode, a file in `dev.files_modified` that is absent from `git diff <baseline_head_sha>` AND absent from `baseline_dirty_snapshot` should NOT be staged. A WARNING line should be printed and the file removed from the candidate set.
- **Gap**: Current code stages provenance_anomaly files (warn-but-stage). Required behavior is warn-and-skip (warn-and-exclude-from-staging for BULK=false).

## Requirements (MoSCoW)

### Must Have

- In BULK=false mode, files in `dev.files_modified` that are absent from `git diff <baseline_head_sha>` AND absent from `baseline_dirty_snapshot` must be excluded from the staging candidate set (not staged).
- A WARNING must be printed for each such skipped file: `WARNING: provenance_anomaly — <path> claimed by dev.files_modified but absent from git diff --name-only <baseline_head_sha>; skipping staging of this file (BULK=false)`.
- The commit must still proceed for files that ARE in the diff (do not abort the whole commit).

### Should Have

- The BULK=false warn-and-skip behavior is documented inline in the provenance filter section so the distinction from BULK=true behavior is explicit.

### Won't Have

- Changes to BULK=true path (BULK=true already uses git status as staging authority; the provenance filter there is enrichment-only and correct as-is).
- Changes to agents/dev.md, agents/qa.md, or commands/dev.md (items 1, 2, 4 already implemented).
