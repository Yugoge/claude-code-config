# BA Specification: /do direct-development cannot be closed via /close normal path

**Request ID**: 20260517-155838
**Created**: 2026-05-17T16:10:00Z
**Tier**: SMALL

## Goal

When the user develops a simple ticket directly with `/do` (bypassing `/dev`), `/close` fails both at task-id resolution and at artifact preflight. Identify the two failure points precisely, and fix them with the minimum correct change: (1) a doc cross-reference in `commands/do.md` and (2) a forced-path logic fix in `commands/close.md` so that `/close --force` with a timestamp argument does not require `qa-report-<task-id>.json` to exist.

## Context

The user's normal workflow for complex work is `/dev` → `/close` → `/commit`. For simple tickets the user uses `/do` to unlock direct development, then tries `/close`. `/do` only writes `/tmp/claude-orchestrator-consent-<session-id>.flag`; it does not create any dev cycle artifacts. `/close` is designed to operate against a `/dev` cycle artifact chain and fails when that chain is absent.

## Edge Cases & Risks

- **Forced path uses same resolution as normal path (verified at close.md:64-65)**: The forced path says "Resolve the task-id from the remaining argument using the same Task-id resolution rules below." This means timestamp form currently runs through the same code that verifies qa-report existence. The fix must branch the timestamp resolution specifically for the forced path.
- **Path form is already safe**: `/close docs/dev/ticket-<task-id>.md --force` only requires the ticket `.md` to exist (path resolution only verifies the named file). This form works today even before R1. The R1 fix only addresses the timestamp convenience form.
- **If no ticket exists at all**: The user has no `ticket-<task-id>.md` because they bypassed BA entirely. In this case even the path form fails. The doc in do.md must clarify this edge case: the user should either (a) create a minimal ticket file first, or (b) after R1 is fixed, use `/close <any-id> --force` with a self-chosen task-id. This is an edge case, not a regression blocker.
- **close.md is not injected by body (`disable-model-invocation: true`)** — NOT a concern here. close.md is a slash command read by the orchestrator, not a subagent prompt; the `disable-model-invocation` frontmatter prevents self-invocation, not reading. The file's workflow body is fully active. (do.md's `disable-model-invocation` means its body is not injected into agent context — that's why any guidance added there must use the section above the hook-only note, OR be a separate visible section.)
