## Implementation Note

`LOCK_TIMEOUT="${LOCK_TIMEOUT:-10}"` intentionally permits external environment override (env-override pattern is the mandated form per spec). Always pass the value as `"${LOCK_TIMEOUT}"` (double-quoted) in the `flock` call to prevent word splitting. Do not add any further parameterization or configuration.

## Edge Cases & Risks

- Word splitting: quoting `"${LOCK_TIMEOUT}"` in the `flock` call is mandatory; bare `$LOCK_TIMEOUT` would fail if the value ever contains whitespace
- No regression risk: the default value `10` is identical to the current hardcoded value; existing behavior is preserved
