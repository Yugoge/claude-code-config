## Root Cause Analysis

**Symptom**: Playwright `connectOverCDP()` hangs ~4s+ per `Runtime.evaluate` when attaching to a wedged Chrome page target.

**Root Cause**: A single Chrome page target wedges in renderer-side futex-wait. The Chrome browser process itself stays healthy, but `connectOverCDP()` auto-attaches to ALL existing page targets at connect time and hangs on the dead one. The persistent Chrome model means dead pages accumulate over time. No journald event fires for the wedge (it is a hang, not a crash), so service-level health checks cannot detect it. Concurrent MCP sessions cannot blindly close each other's pages.

**Root Cause Commit**: N/A — pre-existing architectural choice (persistent Chrome model + Playwright auto-attach semantics) rather than a regression. The fix is additive (new wrapper logic) not a revert.

**Timeline**: Hang manifested 2026-05-19 with `localhost:8096/generate` page wedging Playwright connect. Prior 14-day crash history: 1 V8 SIGTRAP (2026-05-17, post 9h CPU + 446M swap), 1 system-wide OOM (2026-05-14), both unrelated.

## Implementation
