## Goal

Fix .gitignore coverage and remove tracked/committed temporary files (screenshots, workflow state, secrets, lockfiles) across 10 repositories so that no transient artifacts appear in future commits. All 10 repos are private — no public exposure risk.

## Context

## Edge Cases & Risks

- **knowledge-system staged files**: 5 files are staged including private data indices. If committed before .gitignore fix, they land in history. Dev must run `git reset HEAD` and fix .gitignore BEFORE allowing any commit.
- **travel-planner .bak files are tracked**: the live data files exist without .bak extension. Deleting .bak is safe — originals without .bak extension exist.
- **budget-management data/decisions/**: 3 arbiter decision JSON files are tracked and MUST STAY tracked — they are non-regenerable.
- **happy lockfiles**: both bun.lock and package-lock.json to be deleted (Q2 resolved) — dev should confirm yarn.lock is present or will be regenerated before deleting.
- **dot-claude docs/ deletion cascades**: 383 files will be untracked. This includes docs/reference/ and docs/incidents* which are no longer in git after this operation. User confirmed intentionally.
- **orchestra .codex pattern mismatch**: existing `.codex/` in .gitignore only covers a directory, not the empty file `.codex`. Adding bare `.codex` entry covers the file; both entries together cover any future directory too.
- **application-assistant null/ physical delete**: use `rm -rf null/` — the directory has a nested subdirectory. git rm --cached must precede or accompany physical delete.

## Acceptance Criteria

### AC1: .gitignore patterns verified per repo
- GIVEN each repo's .gitignore has been updated
- WHEN `git check-ignore <filename>` is run for each newly-added pattern
- THEN exit code 0 (file is ignored) for every pattern entry

### AC2: Tracked temp files removed from index
- GIVEN `git rm --cached` has been run for all identified files
- WHEN `git ls-files | grep -E "(workflow-[a-f0-9-]+\.json|\.codex|\.playwright-mcp|node_modules)"` is run in each repo
- THEN output is empty (no transient files in index)

### AC3: Screenshots deleted from disk
- GIVEN deletion commands ran for PNG/JPG files in budget-management, travel-planner, multi-asset-portfolio, happy
- WHEN `ls *.png *.jpg` is run at each repo root
- THEN no screenshot files present (only intentional images if any)

### AC4: .env untracked in travel-planner
- GIVEN `git rm --cached .env` ran in travel-planner
- WHEN `git ls-files .env` is run
- THEN output is empty

### AC5: knowledge-system private data not in index
- GIVEN `git rm --cached -r chats/ learning-materials/ knowledge-base/ node_modules/` ran
- WHEN `git ls-files chats/ learning-materials/ knowledge-base/ node_modules/` is run
- THEN output is empty

### AC6: Staged private files not committed
- GIVEN knowledge-system had staged files including private data
- WHEN `git diff --cached --name-only` is run after cleanup
- THEN only legitimate files (workflow hooks, non-private) remain staged, or staging area is clean

### AC7: dot-claude docs/ fully untracked
- GIVEN `git rm -r --cached docs/` ran in dot-claude and `docs/` added to .gitignore
- WHEN `git ls-files docs/` is run in dot-claude
- THEN output is empty

### AC8: happy lockfiles deleted and untracked
- GIVEN bun.lock and package-lock.json deleted and untracked in happy
- WHEN `git ls-files bun.lock package-lock.json` is run in happy
- THEN output is empty
- AND `ls /root/happy/bun.lock /root/happy/package-lock.json` returns "No such file"

### AC9: rednote-mcp INDEX.md gitignored
- GIVEN `INDEX.md` added to rednote-mcp .gitignore
- WHEN `git check-ignore -v INDEX.md` is run in rednote-mcp
- THEN exit code 0 (file is ignored)

### AC10: application-assistant null/ and root data files deleted
