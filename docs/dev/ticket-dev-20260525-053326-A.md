## Goal

Add a `_SCRIPT_INTERPS` branch in `_process_segment()` so that unquoted word-kind argument tokens after the command word are stripped (treated as data, not shell) when the command is a non-shell script interpreter (python3, node, perl, etc.). This eliminates the false positive class where `python3 cleanup.py rm /tmp/old` triggers the `rm`-block hook erroneously.

## Context

`pretool-bash-safety.sh` calls `bash_context_strip.py` to produce `COMMAND_CONTEXT_STRIPPED`, then four danger-token regex patterns run against the stripped text. The stripping already handles: quoted args (single/double), shell interpreter -c payloads (via `_process_shell_interp`), heredoc bodies for non-shell consumers (via `_process_heredocs`), and shell comments. The remaining gap: in `_process_segment()` lines 299-312, tokens with `kind == 'word'` fall through to `out.append(text)` verbatim. For non-shell interpreter commands this keeps positional argv (script name, subcommands, path args) visible, triggering false positives.

## Edge Cases & Risks

- **Process substitution `<(rm ...)`**: The tokenizer stop chars do not include `(`, so `<(rm` tokenizes as a single `word` token. If stripped, the hook's `(^|[ \t;|&(])rm\s` pattern (which uses `(` as a legal left-boundary) would miss `rm`. Fix: preserve word tokens containing `<`, `>`, `(`, `)`, or backtick. (Codex finding 1 â€” verified correct.)
- **Versioned interpreters `python3.12`**: Plain set membership `cmd_word in _SCRIPT_INTERPS` misses these. Mitigated by `_is_script_interp()` helper with `re.match(r'^(python|node|nodejs|perl|ruby|php)\d*(\.\d+)*$', word)`. (Codex finding 4 â€” in scope minor.)
- **Wrapper chains `env python3 script.py rm x`**: `_find_command_word()` already unwraps `env` via `_WRAPPER_PREFIXES` and returns `python3` as cmd_word â€” no additional work needed. (Codex finding 3 â€” already handled.)
- **`DANGER_COMMANDS` in cmd position `rm file1 file2`**: `DANGER_COMMANDS` branch fires before the `_SCRIPT_INTERPS` branch in `_process_segment()` â€” no interference.
