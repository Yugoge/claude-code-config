## Edge Cases & Risks

- **CJK Unicode escapes in Markdown**: Unicode escapes (`之前`) are NOT decoded by Markdown
  renderers or LLM prompt parsers at runtime. For .md agent files, the correct fix is the pointer form
  "(Chinese equivalents: see docs/dev/ticket-20260527-132200.md)" — not Unicode escape sequences.
  Python `.py` files DO decode `"之前"` as the actual CJK character at parse time.
- **agents/graphify.md YAML front-matter**: The description field in the YAML front-matter (line 3)
  contains "Step 7.5" — this is a YAML string value, not a Markdown heading, but it is still a
  prose decimal step reference that Standard 4 flags.
- **AC8 test update**: The existing test_AC8 invokes graphify-enrich.py WITHOUT --context-file.
  The new clause (patching context.json in nil-map path) requires a test that passes --context-file.
  Codex proposes adding this as a new assertion in the existing test function or a sibling test.
- **Fix 5 post-dispatch line**: commands/dev.md:703 says "When graphify completes (or is skipped),
  check .../graph-summary.json". When sentinel was absent (dispatch skipped), graph-summary.json
  is never written. The post-dispatch check must be conditional on sentinel having existed.
- **Trap (spec Section 8)**: venv path inside command .md files is `${CLAUDE_PROJECT_DIR}/venv/bin/activate`,
  not `~/.claude/venv/bin/activate`. The latter is for hook scripts that run from $HOME.
