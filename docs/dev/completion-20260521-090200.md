## Requirement

**Original**: Investigate which hook blocks `/tmp/xlsx-venv/bin/python /root/orchestra/docs/dev/find_red_cells.py > /root/orchestra/docs/research/case-3` despite a valid `/allow` sentinel grant; fix the gaps so this type of command is correctly honored.

**Clarified**: Three independent gaps identified through direct Python execution. PRIMARY: `pretool-tool-policy.py` (matcher `*`, fires for all tools) extracts the Bash redirect target as a Write path and calls `is_allowed('ba', 'Write', '/root/orchestra/docs/research/case-3')` which returns `(False, 'not in allowed_write_path_prefixes')` — this is the hook that actually produced the block. SECONDARY: `pretool-bash-safety.sh:483` wraps the entire sentinel block in `if [ "$IS_SUBAGENT" != "1" ]`, so for subagent contexts the sentinel is never consulted. TERTIARY: task ID writer (`userprompt-consent-allowlist.sh`) uses `CLAUDE_TASK_ID:-SID` (session_id fallback) but reader (`pretool-bash-safety.sh`) prefers `task_id` from PreToolUse JSON first, causing key mismatch when `CLAUDE_TASK_ID` is unset. The prior spec (7 QA objections) had investigated only `pretool-bash-safety.sh` and missed the primary blocker entirely.

## Implementation

### Approach

Three targeted fixes (M1–M3) plus one Should Have (S1):

- **M1** (PRIMARY blocker): Added narrow write-path prefix `*/docs/research/` to `ba.allowed_write_path_prefixes` in `policies/tool-policy.v1.json`, unblocking `pretool-tool-policy.py` for redirect targets in `/root/orchestra/docs/research/`.
- **M2** (SECONDARY blocker): Removed the `IS_SUBAGENT != 1` guard at former line 483 in `pretool-bash-safety.sh`, allowing the structured sentinel block to be consulted for subagent contexts. The IS_SUBAGENT guard at line 145 (legacy pattern-string grant firewall) was preserved unchanged.
- **M3** (TERTIARY gap): Aligned task ID reader fallback in `pretool-bash-safety.sh` to use `d.get('session_id','')` directly (same source as writer's `SID`), eliminating the mismatch scenario.
- **S1** (CF-1 fix): Added early exit in `userprompt-consent-allowlist.sh` when `is_regex=true` — regex grants no longer write a sentinel with `op:"*"` that CF-1 would suppress via the legacy path.
- Additionally added `Bash(/tmp/*-venv/bin/python*)` to `settings.json` allow list per dispatch acceptance criterion.

Both hooks (pretool-bash-safety.sh and pretool-tool-policy.py) must both exit 0 for a Bash command to proceed; M1 alone or M2 alone would be insufficient for subagent contexts.
