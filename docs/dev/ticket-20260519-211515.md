CAS is **mandatory** for every invocation; the flag only changes whether CAS uses the caller-supplied expected value or a self-snapshotted value:

  **Mandatory-CAS protocol (every invocation, under the existing `flock 9`)**:

  1. Acquire `flock -w 10 9` on the agent-scores lockfile (unchanged from pre-iter behavior — flock is at the existing established line in score-update.sh).
  2. **Snapshot phase**: read `current_score = agents[agent]["score"]` from `agent-scores.json`. If `--expected-prev-score` is supplied, set `expected = <supplied>`; else set `expected = current_score` (self-snapshot).
  3. **CAS check phase**: if `current_score != expected`, exit 3 (`CAS_CONFLICT`) with stderr `CAS_CONFLICT: expected=<expected>, current=<current_score>, agent=<agent>`. NO write happens.
  4. **Re-read-before-write phase (drift detection)**: re-read `re_read_score = agents[agent]["score"]` from disk. If `re_read_score != current_score`, exit 3 (`CAS_CONFLICT_DRIFT`) — this catches the case where another process wrote between snapshot and write. (In practice flock prevents this; the re-read is a belt-and-suspenders.)
  5. **Write phase**: compute `new_score = current_score + delta`, write `agents[agent]["score"] = new_score` atomically via tempfile + rename (the existing atomic-write guard at the established line in score-update.sh).
  6. **Lifecycle-emit phase**: invoke `bash scripts/lifecycle-log.sh --event score_update --payload '{"agent":..., "event":..., "prev_score":..., "new_score":..., "delta":..., "reason":...}'`.
  7. Release flock; exit 0.

  **Backward-compat attestation**: existing callers omit `--expected-prev-score` and continue to function — they just use the self-snapshot mode. The CAS race-detection observable in AC11 requires both callers to pass the same `--expected-prev-score` value (so the second caller's expected won't match the first caller's post-write value).

  **Single-writer assumption documented**: in the self-snapshot mode (no `--expected-prev-score`), the race-detection guarantee is weaker — only the re-read-before-write check (step 4) can detect concurrent writers, and flock makes that vanishingly improbable for in-process callers. The documented contract: callers that genuinely care about cross-process race detection MUST pass `--expected-prev-score`. The default self-snapshot mode honors backward-compat AND the user requirement "writes use Compare-And-Swap on prev_score" (the CAS check at step 3 always runs; the only variable is the source of `expected`).

  **Why option (c)**:
  - Option (a) (unconditional CAS, REQUIRED `--expected-prev-score`) breaks every existing caller of `score-update.sh`. Quantified blast radius: per QA's cross-check `scripts/score-update.sh line count = 183`; callers in `commands/close.md`, hooks, and overnight scripts all omit the flag. Breaking signature is unbounded migration cost for a harness-internal improvement.
  - Option (b) (opt-in CAS + amended AC11 + migration plan) preserves backward-compat but leaves AC11's race-detection guarantee unenforceable for real production callers (only the test fixture sees it). The user's R9 says "writes use Compare-And-Swap on prev_score" as the universal write semantic; opt-in CAS contradicts the user requirement.
  - Option (c) (this option) preserves backward-compat (existing CLI signature unchanged) AND makes CAS mandatory on every write (every invocation runs steps 2-4). The trade-off: the self-snapshot mode is a weaker race-detection guarantee than caller-supplied expected — but the user requirement is satisfied (CAS runs on every write) and AC11's observability is preserved (test fixture passes `--expected-prev-score` to both callers to make the race-detection deterministic). Quantified residual risk: in production, two concurrent callers without `--expected-prev-score` are serialized by flock and BOTH succeed (because each self-snapshots its own pre-state). This matches what the prior cycle's `flock`-only design already delivered — no regression — but with the addition of the lifecycle-log audit trail (AC10) that makes any concurrent-write pattern observable in retrospect.
