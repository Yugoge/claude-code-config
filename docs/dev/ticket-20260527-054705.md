## Technical Hints

- **Item A**: The two locations to modify in `agents/changelog-analyst.md` are:
  1. Lines 124-127: Replace the `WARNING: no dev-report found... falling back to stage-all` with `ABORT: no dev-report found... failure_code: scope_violation`. Exit without staging.
  2. Lines 329-340: Add a BULK=false guard to the orphan handling. Orphan commit is appropriate in BULK=true (subsystem grouping needs a catch-all); in BULK=false, orphans indicate cross-session contamination and must be warned+skipped.
- **Item B**: The `normalize_ansic_token()` function goes inside the Python helper between lines 888 and 890 (after the `WRITE_ACTION_FLAGS` set definition, before the token loop). It should:
  1. Use `re.sub` to find `\\x[0-9a-fA-F]{2}` and replace with `chr(int(match[2:], 16))`
  2. Handle `\\0[0-7]{1,3}` (leading-zero octal) and `\\[0-7]{1,3}` (bare octal)
  3. Handle `\\u[0-9a-fA-F]{4}` and `\\U[0-9a-fA-F]{8}`
  4. Handle standard C escapes: `\\n` -> `\n`, `\\t` -> `\t`, `\\r` -> `\r`, `\\a` -> `\a`, `\\b` -> `\b`, `\\f` -> `\f`, `\\v` -> `\v`, `\\\\` -> `\\`, `\\'` -> `'`
  5. Wrap in try/except and fail-closed (return original token on decode error)
  6. Apply to ALL tokens, not just `$`-prefixed ones (shlex may produce reassembled tokens without `$` prefix)
  7. The existing `$-strip` defense at lines 898-903 should be KEPT as a secondary defense layer
- **Item C**: Simple find-and-replace: `bulk-commit-auth-flag-write` -> `bulk-commit-sentinel-write` in all 16 files. No structural changes needed.
