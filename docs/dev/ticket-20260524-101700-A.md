## Goal

The previous fix for bash-safety false positives (task 20260522-080646-A) introduced a bypass: `bash_context_strip.py` must NOT strip the quoted arguments of commands that are themselves danger tokens (`killall`, `pkill`, `kill`, `rm`, `mv`). These arguments ARE the dangerous payload the hook rules check against. The fix is a targeted DANGER_COMMANDS exclusion in `_process_segment()`. The on-disk implementation is correct and both source files are committed. The sole remaining deliverable is registering AC17, AC18, and AC19 in `tests/generated/20260522-080646-A/manifest.json` `active_tests[]` — these test files exist and pass but are orphaned from the manifest.

## Context

Cycle 20260522-080646-A added `hooks/lib/bash_context_strip.py` to strip quoted string arguments from non-shell commands (e.g., `codex exec "...killall...claude..."`) so that documentation text in LLM CLI arguments does not trigger the hook's danger-token rules. QA iter2 confirmed the primary incident case (AC13) was fixed but discovered a bypass: the stripper was treating `killall`, `pkill`, and `kill` as non-shell binaries and blanking their quoted arguments. `killall "happy"` became `killall ""`, which no longer matched the pattern `(killall|pkill)\s+.*(happy|claude|docker)`, so the hook exited 0 instead of 2.

The on-disk code **already contains the correct fix**: a DANGER_COMMANDS branch in `_process_segment()` at lines 264-273 that unquotes (rather than blanks) arguments to danger commands, preserving their content for hook pattern matching. `killall "happy"` → `killall happy` → pattern matches → exit 2. The unit tests for this behavior (`TestDangerCommandArgPreservation`, 4 cases) are in `hooks/tests/test_bash_safety_context.py`; 30 unit tests pass total.

## Edge Cases & Risks

- **Risk 1**: The unquoting in the DANGER_COMMANDS branch (`text[1:-1]`) must not introduce a new bypass for single-char quoted strings. `kill ""` (empty quoted arg) → `kill ` (trailing space) — the kill-signal pattern `(^|[ \t;|&])(kill)[ \t]+-` requires `-` after whitespace, so `kill ` alone does not match. No bypass introduced.
- **Risk 2**: `rm "$(dangerous)"` — the DANGER_COMMANDS branch at line 269 only unquotes `kind in {"single", "double"}`. Double-quoted strings containing `$()` substitutions are also in the `double` kind; unquoting them exposes the `$(...)` for subsequent pattern matching. This is safe — the substitution content remains visible.
- **Risk 3**: Ensure `killall happy` (unquoted, bare) still matches — verified: the unquoting path does not strip bare words, only quoted tokens.
- **Risk 4**: Manifest update must be append-only. Dev must NOT rewrite the entire manifest — only append three new entries to `active_tests[]`. The 16 existing entries must be preserved unchanged. Schema version must remain `1.1`.
