## Implementation

### Approach

Two-part fix targeting the root cause: (1) `bash_write_targets.py::_resolve_path()` expands only `$HOME`, `~`, and `$CLAUDE_PROJECT_DIR` — a `tee "$CODEX_OUT"` call yields the literal token `$CODEX_OUT` to the policy checker, matching no allowed prefix; (2) `docs/codex/` was absent from all 11 consultation-role write-path allowlists. The fix adds the prefix to the policy and replaces the opaque variable tee forms with inline `$CLAUDE_PROJECT_DIR/docs/codex/$CODEX_OUT_TASK_ID/$CODEX_OUT_ROLE.txt` paths which `_resolve_path()` can expand.

**Note on prior attempts**: Two prior BA attempts (ticket `fa9f07` and an earlier 090100 revision) had enumeration errors (9 roles instead of 11, dev incorrectly included, SKILL.md omitted) and AC weaknesses (grep-fragile set check, permitted opaque tee forms). The final spec incorporated all Codex adversarial-review findings (CF-1 through CF-5) into set-based JSON-semantic ACs.
