# BA Specification: Wrap bare python3 heredoc calls with venv activation

## Goal

Wrap the two bare `python3` heredoc invocations at lines 151 and 194 of
`tests/score-lifecycle-contract/test-lifecycle-cas.sh` with venv activation,
matching the project's Standard 3 requirement (no bare `python3` calls).

## Context

The style inspector flagged two `introduced_in_diff: true` Standard 3 violations
in the test file. A bare `python3` at lines 151 and 194 may resolve to the system
Python rather than the project venv, causing import failures. Line 86 of the same
file already uses the correct pattern inside an `if` condition; lines 151 and 194
need the same venv activation wrapped in a standalone subshell.

## Setup / Environment

- **applicability**: N/A
- **reason**: non-UI -- CLI; cycle does not produce (1) rendered UI changes, (2) browser interaction, (3) Playwright invocation, (4) screenshot evidence, or (5) any change to user-triggered code paths

## Affected Location

File: `tests/score-lifecycle-contract/test-lifecycle-cas.sh`
Lines: 151 and 194 (each is the opening line of a python3 heredoc block)

Existing pattern at line 86 (inside `if` condition — for reference only):
```bash
if source ~/.claude/venv/bin/activate && python3 - "$last_line" <<'PYCHECK'
```

Required shape for lines 151 and 194 (standalone subshell — NOT an `if` condition):
```bash
( source ~/.claude/venv/bin/activate && python3 - "$FAKE_SCORES" <<'PYEOF'
...heredoc body unchanged...
PYEOF
)
```

The heredoc delimiter line (`PYEOF`) is unchanged. A closing `)` on its own line
immediately follows the `PYEOF` line.

## Non-Goals

- No other files are touched.
- No changes to heredoc body content.
- No changes to test logic, assertions, or surrounding bash structure.

## Acceptance Criteria

### AC1: Venv activation present at both call sites

GIVEN `tests/score-lifecycle-contract/test-lifecycle-cas.sh` with the fix applied
WHEN dev runs:
  1. `bash -n tests/score-lifecycle-contract/test-lifecycle-cas.sh` (syntax check)
  2. `grep -c 'source.*venv.*activate.*&&.*python3' tests/score-lifecycle-contract/test-lifecycle-cas.sh`
  3. `grep -cP '^python3' tests/score-lifecycle-contract/test-lifecycle-cas.sh`
THEN (1) exits 0; (2) outputs `3` (line 86 pre-existing + lines 151 and 194 after fix); (3) outputs `0` on stdout — no bare `python3` calls remain at line start. Note: cmd3 (`grep -cP '^python3'`) exits with code 1 when count is 0; stdout '0' is the correct result. A grep exit code of 1 for zero matches is expected behavior and is not a failure.

### AC2: Existing tests still pass

GIVEN the fix is applied and the test harness dependencies are present
WHEN dev runs `bash tests/score-lifecycle-contract/test-lifecycle-cas.sh` from repo root
THEN the script exits 0 (all CAS tests pass)
