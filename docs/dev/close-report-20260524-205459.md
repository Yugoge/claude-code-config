## Summary

arch-3 phase 2: single-line redirect reorder at `hooks/userprompt-tmpfs-pressure.sh:129`.

Change: `} 9> "$LOCK_FILE" 2>/dev/null` → `} 2>/dev/null 9> "$LOCK_FILE"`

Rationale: Bash evaluates redirections left-to-right. The prior order opened fd-9 to
LOCK_FILE before stderr was redirected, so an EISDIR failure leaked one line to real
stderr. The fix places `2>/dev/null` first, silencing any fd-9 open failure before it
reaches the terminal.
