# BA Specification: Fix `python` → `python3` and `CLAUDE_SESSION_ID` fallback

**Request ID**: dev-20260527-063758-T4
**Created**: 2026-05-27T06:37:58Z
**Tier**: SMALL

## Goal

Fix two co-occurring invocation failures that block `/commit` and `/push` on this system:
(A) the `python` command does not exist — only `python3` is installed; (B) three scripts
abort because `CLAUDE_SESSION_ID` is always unset — only `CLAUDE_CODE_SESSION_ID` is set.

## Context

On this Ubuntu 22.04+ system, `python` is not symlinked to `python3`.
`commands/commit.md` Step 5 invokes scripts using bare `python`, which exits 127.
The workaround of passing `--sid` to `write-commit-grant.py` covers one script but
not `execute-push.py` (which reads `CLAUDE_SESSION_ID` with no CLI override) or
`write-bulk-commit-sentinel.py` (same). The scripts should resolve the session ID
from whichever env var is actually set.

## Requirements (MoSCoW)

### Must Have
- Replace `python` with `python3` at all three locations in `commands/commit.md` where it invokes scripts via bash
- In `commands/commit.md` lines 83-84 (prose abort instructions in Step 5): update both the BULK=true prose (line 83) and the BULK=false prose (line 84) so the abort condition recognizes `CLAUDE_CODE_SESSION_ID` as an alternative — the orchestrator reads and executes this prose, so a single-var abort on `CLAUDE_SESSION_ID` fires before the scripts are ever invoked
- In `commands/push.md` lines 123-127 (prose `SESSION_ID` assignment and abort): update `SESSION_ID="${CLAUDE_SESSION_ID}"` to use a fallback expression (`SESSION_ID="${CLAUDE_SESSION_ID:-${CLAUDE_CODE_SESSION_ID:-}}"`), and update the abort message to name both vars — this is required to unblock `/push`, as push.md aborts before dispatching execute-push.py
- In `scripts/write-commit-grant.py` `_resolve_sid()`: fall back to `CLAUDE_CODE_SESSION_ID` when `CLAUDE_SESSION_ID` is empty
- In `scripts/execute-push.py` `main()` Step 2: fall back to `CLAUDE_CODE_SESSION_ID` when `CLAUDE_SESSION_ID` is empty
- In `scripts/write-bulk-commit-sentinel.py` `_resolve_sid()`: fall back to `CLAUDE_CODE_SESSION_ID` when `CLAUDE_SESSION_ID` is empty
- When `CLAUDE_SESSION_ID` IS set, it takes priority (backward-compatible)
- When BOTH env vars are unset, the existing abort behavior (exit 2) must remain

- Fixing `dev-command.md:695` (permission-string template, not an actual invocation)
- Adding `python3` shebang to scripts (already have `#!/usr/bin/env python3`)
