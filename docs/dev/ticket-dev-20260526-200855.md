# BA Specification: Fix execute-push.py CWD dependency via --repo-root

**Request ID**: dev-20260526-200855
**Created**: 2026-05-26T20:08:55Z
**Tier**: SMALL

## Goal

Add an optional `--repo-root` CLI parameter to `execute-push.py` so the script operates on the correct git repository regardless of the caller's CWD. Update `commands/push.md` to capture `REPO_ROOT` in Step 2 and pass it via `--repo-root` in Step 5.

## Context

`execute-push.py` runs `git rev-parse HEAD` (step 9, line 220) without specifying a CWD. During a prior `/push` invocation the orchestrator ran from `/dev/shm/dev-workspace/dot-claude` (the nested `.claude` repo) instead of `/root` (the main repo). Git resolved the wrong HEAD hash (`0e58363d` vs `2eecfa87`), the drift check at line 226 failed, and the push was aborted. A one-time `cd /root` workaround was used but not codified.

The fix is two-part: (1) add `--repo-root` to `execute-push.py` so `os.chdir()` is called at startup, making all git operations (and the inherited `push.sh` via `os.execv`) use the correct directory; (2) update `commands/push.md` to capture `REPO_ROOT` in Step 2 and pass it in Step 5.

## Requirements (MoSCoW)

### Must Have

- `scripts/execute-push.py`: add `--repo-root` optional CLI argument (argparse, default `None`).
- `scripts/execute-push.py`: when `--repo-root` is provided, call `os.chdir(args.repo_root)` immediately after arg parsing (before Step 2 session_id check and all subsequent git operations). Wrap in `try/except OSError`; on failure print stderr and `return 2` before any grant file is touched.
- `scripts/execute-push.py`: update module docstring Usage block to show `[--repo-root <REPO_ROOT>]`.
- `commands/push.md` Step 2: capture `REPO_ROOT` using the canonical `realpath` form — `REPO_ROOT=$(realpath "$(git rev-parse --show-toplevel)")` — as a separate variable, then derive `REPO_HASH` from it — `REPO_HASH=$(printf '%s' "${REPO_ROOT}" | sha256sum | cut -c1-16)`. Using `realpath` at capture time ensures `REPO_ROOT` is already canonical, so `REPO_HASH` needs no second `realpath` call. This ensures `REPO_ROOT` and `REPO_HASH` are bound to the same resolved root.
- `commands/push.md` Step 5: pass `--repo-root "${REPO_ROOT}"` in both the non-auto and auto invocation blocks. `--repo-root "${REPO_ROOT}"` must appear as a continuation line immediately before `--auto` in the auto block.
- `commands/push.md` Step 5: remove `source venv/bin/activate &&` prefix from both invocation blocks. Invoke `python3 ~/.claude/scripts/execute-push.py` directly, consistent with the existing prose note at push.md line ~208 ("python3 is used directly (no venv activation needed)").

### Should Have

- Error message on invalid `--repo-root` should name the path and the OS error, e.g. `execute-push: cannot chdir to '/bad/path': [Errno 2] No such file or directory. Aborting.`

## Edge Cases & Risks

- **Invalid path**: `--repo-root` points to a nonexistent or non-directory path. `os.chdir()` raises `OSError`; must return exit 2 without touching any grant file.
- **Not a git repo**: `--repo-root` is a valid directory but not a git repo. `git rev-parse HEAD` fails with `CalledProcessError`; the existing handler at line 223 already covers this and prints an error.
- **REPO_HASH consistency**: if Step 2 is updated to derive `REPO_HASH` from `REPO_ROOT` (via `realpath "${REPO_ROOT}"`), the hash must remain identical to the current computation — `realpath` on the same directory yields the same canonical path.
- **push.sh CWD inheritance**: `os.execv` replaces the process image; the CWD set by `os.chdir()` is inherited by `push.sh`. This is the desired behavior and closes the full surface area.
