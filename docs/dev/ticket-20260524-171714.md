## Requirements (MoSCoW)

### Must Have

- **M1 (arch-6)**: Create `hooks/pretool-gitignore-preflight.py` — a PreToolUse hook registered in `settings.json` under a `matcher: "Agent"` block. The hook reads the dev-report path from the Agent tool's prompt JSON (field `Dev report file: docs/dev/dev-report-<task-id>.json` pattern). If no dev-report path is identifiable in the prompt, the hook is a no-op (exit 0). If the dev-report path exists, it reads `dev.files_modified` and `dev.files_created`, then checks each path against `.gitignore` using `git check-ignore --no-index --quiet -- <path>` (per-path exit-code check). Any path that is gitignored AND lacks a top-level `gitignore_waiver` value that is non-null and non-empty causes the hook to exit **code 2** (`sys.exit(2)`, consistent with all existing PreToolUse blocking hooks in this repo) with a human-readable BLOCKED message naming the offending files. If the dev-report path is absent on disk, the hook is a no-op (exit 0) — it must not block pre-dev dispatches.

- **M2 (arch-3)**: In `hooks/userprompt-tmpfs-pressure.sh`, replace both counter-write lines inside the flock block with a failure-aware form. The critical write at line 118 (`echo $((n + 1)) > "$COUNTER_FILE"`) must become: `if ! printf '%s\n' "$((n + 1))" 2>/dev/null > "$COUNTER_FILE"; then printf 'skip'; exit 0; fi; printf 'emit'`. The mtime-refresh write at line 115 (`printf '%s\n' "$n" > "$COUNTER_FILE"`) must also be wrapped: `printf '%s\n' "$n" 2>/dev/null > "$COUNTER_FILE" || { printf 'skip'; exit 0; }`. The `2>/dev/null` is placed before the output redirect so that EISDIR errors (or any stderr from printf itself) are suppressed before reaching the caller — Bash processes redirections left-to-right, so stderr-first is required to silence all error output. In both failure cases, the subshell emits `skip` and exits, causing the parent to silently exit 0 (non-blocking). No lifecycle.jsonl logging in this cycle.

## Edge Cases & Risks

1. **Agent matcher may be too broad** — the `matcher: "Agent"` block fires on ALL Agent dispatches (BA, QA, dev, style-inspector, etc.). If the prompt contains no dev-report reference, the hook must exit 0 without reading any file. The no-op-on-missing-path-in-prompt rule is essential.
2. **Negation rule false positives** — `.gitignore` has `!docs/reference/` and `!logs/lifecycle.jsonl` as negations. Using `git check-ignore -v` output directly would mention negation rules and cause false positives. The per-path `--quiet` exit-code approach (exit 1 = not ignored, exit 0 = ignored) correctly handles negations because git resolves the full chain before exiting.
3. **ENOSPC + flock file** — the lock file itself (`LOCK_FILE`) is created via `9> "$LOCK_FILE"` by the subshell invocation. Under ENOSPC, the lock-file creation may also fail, in which case `flock -x -w 1 9 || { printf 'skip'; exit 0; }` already handles the skip path (the existing flock timeout path covers this).
4. **Both counter-write lines must be patched** — the mtime-refresh write at line 115 (inside the `n >= RATE_LIMIT` branch) also writes to `$COUNTER_FILE`. Under ENOSPC it should also fail gracefully, not crash the subshell.
5. **gitignore_waiver field** — the spec (R1) says the waiver must be in the dev-report. Dev and QA must understand this field exists even before arch-6 lands. The hook must check for `gitignore_waiver` at the top level of the dev-report JSON object (not nested under `dev`).

- **Affected files**:

- **Dev-report path extraction**: the hook receives the Agent tool's input as JSON on stdin with shape `{"tool_name": "Agent", "tool_input": {"prompt": "..."}}`. Extract the prompt via `tool_input = data.get("tool_input") or data.get("toolInput") or {}; prompt = tool_input.get("prompt", "")`. Then extract the dev-report path via regex on the prompt string: `r'docs/dev/dev-report-[A-Za-z0-9._-]+\.json'`. If zero matches, no-op. **Never** access `data.get("prompt")` at the top level — that key is absent in production payloads and will silently return None.
- **gitignore check command**: `subprocess.run(['git', 'check-ignore', '--no-index', '--quiet', '--', path], cwd=repo_root)` — exit code 0 = ignored, exit code 1 = not ignored, exit code 128 = error (treat as not-ignored).
- **Repo root**: derive from `git rev-parse --show-toplevel` or from `__file__`'s parent chain.
- **Counter write pattern (arch-3)**: inside the flock block at line ~118, replace `echo $((n + 1)) > "$COUNTER_FILE"` with:
  ```bash
  if ! printf '%s\n' "$((n + 1))" 2>/dev/null > "$COUNTER_FILE"; then
    printf 'skip'
    exit 0
  fi
  printf 'emit'
  ```
  At line ~115 (mtime refresh), replace `printf '%s\n' "$n" > "$COUNTER_FILE"` with:
  ```bash
  printf '%s\n' "$n" 2>/dev/null > "$COUNTER_FILE" || { printf 'skip'; exit 0; }
  ```
  Note: `2>/dev/null` is placed before `>` in both forms. Bash processes redirections left-to-right; placing stderr suppression first ensures EISDIR (and any other stderr from printf) is silenced before the output redirect is evaluated, so neither stdout nor stderr leaks to the caller under any write-failure scenario.
