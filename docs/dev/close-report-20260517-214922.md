However: the equals form is not the established calling convention in any command file. All fallback bash blocks use `--spec $SPEC_PATH` (space-separated). Risk is low. Not a regression — this code is new.

**Proposed fix** (Codex provided): extend the regex to `(?:--|[–—])spec(?:\s+|=)\S+`. QA notes this is a safe improvement but not blocking.
