# BA Specification: Fix failing test — write-bulk-commit-sentinel.py session-ID fallback

**Request ID**: dev-20260529-081000
**Created**: 2026-05-29T08:10:00Z
**Tier**: SMALL

## Goal

Fix a failing unit test in `hooks/tests/test_bulk_commit_sentinel.py` that incorrectly tests the "no session ID available" path of `scripts/write-bulk-commit-sentinel.py`. The test strips only `CLAUDE_SESSION_ID` from the environment but the writer also resolves via `CLAUDE_CODE_SESSION_ID`, so the test passes unexpectedly when `CLAUDE_CODE_SESSION_ID` is present in the CI/agent environment.

## Context

The user's requirement described a security vulnerability in `pretool-git-privilege-guard.py` where `BLESSED_BRIDGE_RE` allows commits without verifying a bulk-commit sentinel. **That vulnerability was already fixed in commit `41063a1` (2026-05-24)**, which added `_has_bulk_commit_sentinel()` and wired it into `_evaluate_commit()`. The sentinel-check enforcement is fully in place and all relevant behavioral tests pass.

What remains broken is one test: `TestWriteBulkCommitSentinelScript.test_fails_without_session_id` (line 188 of `hooks/tests/test_bulk_commit_sentinel.py`). The test was written before `CLAUDE_CODE_SESSION_ID` was added as a secondary fallback in the writer (`scripts/write-bulk-commit-sentinel.py:56`). In environments where `CLAUDE_CODE_SESSION_ID` is set (all Claude Code agent contexts), the test does not achieve its stated goal of verifying the "both SIDs absent" failure path.
