## Summary

Fixed `pretool-git-privilege-guard.py` so that user grants written by `/allow` are honored
by the privilege guard for all agents (including subagents). Previously, the guard's
`_check_git_allowlist()` only consulted the legacy grant file and never checked the
structured sentinel written by `/allow`, causing all user-authorized commands to still be
blocked at the privilege guard layer even when `pretool-bash-safety.sh` had already allowed
them via the sentinel path.

## Root Cause

`_check_git_allowlist()` was only updated in `pretool-bash-safety.sh` (task 20260521-090200,
M2) but not in `pretool-git-privilege-guard.py`, which is a parallel independent PreToolUse
hook. Both hooks run for every Bash command; both must allow the command for it to proceed.
The fix to bash-safety.sh made no difference because the privilege guard still blocked.

## Implementation

**File modified**: `hooks/pretool-git-privilege-guard.py`

Changes:
- Added `match_sentinel_grant_for_bash_command` to import alongside `read_grant_for_git_command` (line 92)
- Restructured `_check_git_allowlist()` (lines 144-162): sentinel check runs first, before the
  subagent firewall. The subagent firewall remains on the legacy-grant path only, mirroring
  the M2 design decision already implemented at `pretool-bash-safety.sh:484`.

```python
def _check_git_allowlist(command: str, data: dict) -> bool:
    sid = _get_session_id(data)
    if not sid:
        return False
    # Sentinel grant: NOT gated by subagent firewall (M2 decision, task 20260521-090200)
    task_id = os.environ.get('CLAUDE_TASK_ID') or sid
    if match_sentinel_grant_for_bash_command(task_id, command) is not None:
        return True
    # Legacy grant: subagent firewall preserved
    if data.get('agent_id'):
        return False
    return read_grant_for_git_command(command, sid)
```
