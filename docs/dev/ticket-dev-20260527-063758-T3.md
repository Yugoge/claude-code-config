## Goal

Increase `PUSH_ANALYST_GRANT_TTL_SECONDS` from 180 to 600 in `agents/push-analyst.md` so that push-analyst grants remain valid through the full orchestrator -> push-analyst -> orchestrator result-processing -> execute-push.py round trip, which frequently exceeded the previous 180-second window.

## Context

`agents/push-analyst.md` Phase 7 computes `expires_at` using a named constant `PUSH_ANALYST_GRANT_TTL_SECONDS = 180` (line 164). The orchestrator dispatches push-analyst, awaits its completion, processes the result, and then calls `execute-push.py` to validate and consume the grant. This round trip frequently exceeds 180 seconds, causing `execute-push.py` to read an already-expired grant and abort the push. The previous TTL was raised from 120s to 180s (task 20260519-211515 R4 / AC4); that increase proved insufficient.
