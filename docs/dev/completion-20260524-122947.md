## Requirement

**Original**: Root-cause analysis of /push orchestration failures identified two structural defects: (1) Steps 4+5 of /push were being delegated to subagents because the orchestrator exhausted its 3-Bash streak budget on Steps 0-2; (2) /push Step 5 lacked an explicit delegation warning, causing orchestrators to use HOW-prescriptive subagent prompts which caused deletion of the wrong file (Chain-A token instead of Chain-B grant).

**Clarified**: Create scripts/execute-push.py encapsulating Steps 4+5 into a single Bash invocation, and add an explicit delegation warning to commands/push.md Steps 4 and 5.

**Success Criteria**: All 11 ACs from acceptance-criteria-20260524-122947.json pass (verified by QA).

## Root Cause Analysis

**Symptom**: /push failed: subagent deleted push-gate token (Chain-A), push.sh exited 3, and no recovery path existed. TTL mismatch (180s grant, 60s sentinel, 30-90s per dispatch) made multi-agent approaches structurally infeasible.

**Root Cause**: /push Step 5 bash code in commands/push.md required the orchestrator to execute multiple inline bash blocks for Steps 4+5, but no script wrapped this logic. After exhausting the 3-consecutive-Bash budget on Steps 0-2, the orchestrator delegated Steps 4+5 to a subagent. The subagent received HOW-prescriptive instructions (specific file paths to unlink), executed them, but deleted the wrong file. Separately, subagents legitimately refuse to write the Chain-B sentinel (treated as privilege escalation).

**Timeline**: Design gap existed since the /push single-process pattern was first written; first surfaced in production during a /push cycle where orchestrator Bash budget was exhausted.

## Implementation

**Approach**: Created scripts/execute-push.py (335 lines) that encapsulates the entire Steps 4+5 sequence — grant validation, grant consumption, HEAD-drift check, Chain-B sentinel atomic write, and os.execv into push.sh — as a single Python process. Updated commands/push.md Step 4 to redirect to the script (removing orchestrator-imperative bash) and added a hard-to-miss WARNING block in Step 5.

**Files Created**:
- scripts/execute-push.py — single-process grant validator + sentinel writer + push.sh exec
  - Args: --request-id (required), --repo-hash (required), --remote (required), --auto (optional flag)
  - Exit codes: 0/1/2 from push.sh, 1 = grant validation failure, 2 = argument/env setup error
  - Usage: python3 /root/.claude/scripts/execute-push.py --request-id "$REQUEST_ID" --repo-hash "$REPO_HASH" --remote "$RESOLVED_REMOTE"

**Files Modified**:
- commands/push.md — Step 4 converted from orchestrator-imperative bash to reference paragraph; Step 5 adds WARNING block with both delegation-failure reasons (privilege escalation + 60s TTL)

**Key Design Decisions**:
- Script uses stdlib-only (no venv needed) — compatible with existing Bash(python3:*) allow entry
- os.execv used (not subprocess) to maintain single-process semantics for Chain-B sentinel freshness
- Repo-hash validated against computed value before use in file paths (path traversal prevention)
- Blocked verdict exits before grant unlink and before sentinel write (no partial state)
