# BA Specification: Fix Legacy Grant Subagent Leak in /allow Hook Infrastructure

**Request ID**: dev-20260524-205811

**Created**: 2026-05-24T20:58:11Z

## Goal

Remove the subagent-context guard around `consume_grant_for_posttool()` in `hooks/posttool-allowlist-consume.py` so that legacy grant files are always unlinked on terminal result, regardless of whether the executing context is a subagent or the main agent. Also remove the now-dead `is_subagent_context` import and update stale docstrings.

## Context

The `/allow` command (`hooks/userprompt-consent-allowlist.sh`) writes TWO grant files:
1. **Legacy grant**: `/tmp/claude-bash-allowlist-<sid>.json` — `{pattern, is_regex}` — used by `pretool-bash-safety.sh` for main-agent Bash authorization
2. **Sentinel grant**: `/tmp/claude-grants/<task_id>.json` — structured `{task_id, session_id, allowed_operations[], ...}` — used by both main-agent and subagent authorization paths

When a **subagent** executes a Bash command authorized by a sentinel grant (per M2, task 20260521-090200, sentinel grants are honored in subagent context), `posttool-allowlist-consume.py` correctly consumes the sentinel but skips `consume_grant_for_posttool()` due to the `if not is_subagent_context(data):` guard. This leaves the legacy grant file on disk. The main agent can then match and reuse that legacy grant for a second command invocation, bypassing single-use semantics.

2. **FileNotFoundError on second call**: If the legacy grant was already absent (e.g., never written, or already consumed), `consume_grant_for_posttool()` returns `False` silently — no panic. Confirmed at `hooks/lib/allowlist.py:333`.

4. **Concurrent execution**: The existing `fcntl.flock(LOCK_EX)` inside `consume_grant_for_posttool()` ensures atomicity — no race condition introduced.

## Acceptance Criteria

### AC1: Subagent Bash terminal result unlinks legacy grant
- GIVEN a legacy grant at `/tmp/claude-bash-allowlist-<sid>.json` and a matching sentinel at `/tmp/claude-grants/<task_id>.json`
- WHEN `posttool-allowlist-consume.py` receives a Bash PostToolUse stdin payload with `agent_id` present, matching `tool_input.command`, and `tool_response.exit_code=0`
- THEN both the legacy grant file and the sentinel file are absent after the hook runs, and the hook exits 0

### AC2: Main-agent Bash terminal result still unlinks legacy grant (regression)
- GIVEN a legacy grant at `/tmp/claude-bash-allowlist-<sid>.json`
- WHEN `posttool-allowlist-consume.py` receives a Bash PostToolUse stdin payload WITHOUT `agent_id`, matching `tool_input.command`, and `tool_response.exit_code=0`
- THEN the legacy grant file is absent after the hook runs, and the hook exits 0

### AC3: Subagents cannot write new grants (write firewall unchanged)
- GIVEN a UserPromptSubmit payload with `agent_id` present
- WHEN `hooks/userprompt-consent-allowlist.sh` is invoked
- THEN neither `/tmp/claude-bash-allowlist-<sid>.json` nor `/tmp/claude-grants/<task_id>.json` is written, and the hook exits 0 (Step 0 firewall preserved)

### AC4: Sentinel consumed on subagent Bash terminal result
- GIVEN a matching sentinel at `/tmp/claude-grants/<task_id>.json` (no legacy grant)
- WHEN `posttool-allowlist-consume.py` receives a Bash PostToolUse stdin payload with `agent_id` present and `tool_response.exit_code=0`
- THEN the sentinel file is absent after the hook runs; AND the same holds when `tool_response.exit_code=1` (non-zero case)

### AC5: No panic when legacy grant absent
- GIVEN no legacy grant file (absent)
- WHEN `posttool-allowlist-consume.py` receives a Bash PostToolUse stdin payload with `agent_id` present, and a valid sentinel is present
- THEN the hook exits 0, no traceback appears in stderr, and the sentinel is consumed normally

### AC6: Legacy-only subagent PreToolUse authorization remains blocked
- GIVEN only a legacy grant at `/tmp/claude-bash-allowlist-<sid>.json` (no sentinel)
- WHEN `pretool-bash-safety.sh` receives a Bash PreToolUse payload with `agent_id` present and a command matching the legacy grant pattern
- THEN the hook does NOT allow the command via the legacy path (IS_SUBAGENT=1 firewall in `check_and_consume_allowlist` returns 1)

- **Exact changes** (4 edits total):

  2. Lines 87-89: replace the `if not is_subagent_context(data):` guard with an unconditional call: `consume_grant_for_posttool(session_id, tool_name, command)`

- **4th edit — whitespace-normalization atomicity** (required by QA/Codex rejection): When sentinel Bash consumption succeeds (`should_consume = True` in the Bash terminal block of `posttool-allowlist-consume.py`), also unconditionally unlink the legacy grant for `session_id`. This closes the whitespace-normalization divergence case where the sentinel matches (tokenized) but the legacy literal-match does not (e.g., `"git   push"` vs pattern `"git push"`). Add the following immediately after the sentinel-consume call in the Bash terminal result block:

  ```python
  # When sentinel is consumed, also unlink legacy grant unconditionally.
  # This closes the whitespace-normalization divergence case where the
  # sentinel matches (tokenized) but the legacy literal-match doesn't.
  legacy_path = Path(f"/tmp/claude-bash-allowlist-{session_id}.json")
  try:
      legacy_path.unlink()
  except FileNotFoundError:
      pass
  ```

  This ensures two-grant-unlink atomicity: whenever the sentinel is consumed (the authoritative structural match), the legacy grant is also unlinked regardless of whether the literal pattern would have matched the whitespace-normalized command string.
