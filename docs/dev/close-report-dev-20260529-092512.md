**Characterization**: the delta affects ONLY the bare no-operand verb forms. Runtime semantics:
- bare `dd` with no `of=<device>` cannot overwrite a disk (it copies stdin→stdout);
- bare `shred` with no FILE operand exits with `missing file operand` and does nothing.

Neither bare form is capable of destruction. **Every operand-bearing (i.e., actually destructive) invocation still blocks.** The old rule blocking the bare forms was an over-broad match on a harmless fragment, not protection against a destructive command.

QA put the decisive question to Codex: *"give a concrete DESTRUCTIVE command string that now bypasses, or concede it is non-blocking."* Codex re-ran the live hook and replied:

> **CODEX_RULING: NON_BLOCKING** — operand-bearing destructive forms still block; only bare `dd`/`shred` are newly allowed, and they are non-destructive no-op/error forms.
