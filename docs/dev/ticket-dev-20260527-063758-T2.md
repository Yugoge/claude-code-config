# BA Specification: Increase Commit Grant TTL from 10 to 30 Minutes

**Request ID**: dev-20260527-063758-T2
**Created**: 2026-05-27T06:37:58+00:00
**Tier**: SMALL

## Goal

Increase `GRANT_TTL_MINUTES` in `scripts/write-commit-grant.py` from 10 to 30 so that the single-use commit grant survives the full dispatch-to-subagent-commit latency window without expiring.

## Context

`/commit` writes a single-use commit grant (via `write-commit-grant.py`) before dispatching the `changelog-analyst` subagent. The grant is consumed on first use. When dispatch + agent processing takes more than 10 minutes, the grant's `expires_at` passes and `pretool-git-privilege-guard.py` rejects the commit with `grant_expired` — requiring manual revoke + retry. The bulk-commit sentinel (written by `/commit --bulk`) already uses a 30-minute TTL for the same latency reason. The single-use grant should use the same window.

Observed incident (user report 2026-05-27, `docs/dev/user-requirement-dev-20260527-063758.md`): commit grant expired during a slow changelog-analyst dispatch cycle, requiring revoke + rewrite + re-dispatch.
