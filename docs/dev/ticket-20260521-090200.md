## Goal

Identify every PreToolUse hook that blocks `/tmp/xlsx-venv/bin/python /root/orchestra/docs/dev/find_red_cells.py > /root/orchestra/docs/research/case-3` despite a valid `/allow` sentinel grant, and fix the gaps so that a user-issued `/allow` for this command type is correctly honored.

## Context

The `/allow` command writes two grant artifacts: (1) a legacy pattern-string grant at `/tmp/claude-bash-allowlist-<sid>.json`, and (2) a structured sentinel grant at `/tmp/claude-grants/<task_id>.json`. Despite a valid sentinel grant existing, the command was blocked.

**Blocking hook identified (OBJ-1 and OBJ-2 — confirmed by direct Python execution)**:

The PRIMARY blocker is `pretool-tool-policy.py`, which fires for ALL tools (matcher `*` in `settings.json` lines 448-454). For the Bash command with `> /root/orchestra/docs/research/case-3` as redirect target, `extract_bash_write_paths()` at `hooks/lib/bash_write_targets.py:375-399` extracts that path as a Write target (measured: `extract_bash_write_paths('/tmp/xlsx-venv/bin/python /root/orchestra/docs/dev/find_red_cells.py > /root/orchestra/docs/research/case-3')` returns `['/root/orchestra/docs/research/case-3']`). The hook then calls `is_allowed('ba', 'Write', '/root/orchestra/docs/research/case-3')` which returns `(False, 'write target /root/orchestra/docs/research/case-3 not in allowed_write_path_prefixes')` — measured by direct Python execution. This is confirmed at `policies/tool-policy.v1.json:119-140`: `ba.allowed_write_path_prefixes` does not include any `/root/orchestra/` prefix.

For main-agent context (no agent_id): `pretool-tool-policy.py:95-101` calls `resolve_agent_type(data)` which returns `None` → `sys.exit(0)`. No role check runs. The tool-policy hook does NOT block the main agent.

The SECONDARY gap is `pretool-bash-safety.sh`: the structured sentinel block at line 483 is wrapped in `if [ "$IS_SUBAGENT" != "1" ]` — so when `IS_SUBAGENT=1`, the sentinel check is ENTIRELY SKIPPED. The legacy allowlist is also blocked at line 145 (`if [ "$IS_SUBAGENT" = "1" ]; then return 1; fi`). This means that for subagent contexts, even a valid sentinel grant in `/tmp/claude-grants/` cannot unblock the command via `pretool-bash-safety.sh`.
