## Requirement

**Original**: Fix the security vulnerability where `changelog-analyst` can forge an `auto-bulk:` commit message prefix in `BULK=false` mode to bypass the privilege guard's grant verification. The primary bypass (BLESSED_BRIDGE_RE without sentinel) was already fixed in commits 1238f42+41063a1. The remaining gap: an agent could forge a valid bulk-commit sentinel by invoking `write-bulk-commit-sentinel.py` via absolute path or `python3 -c` with a dynamic path, bypassing `pretool-bash-safety.sh` Layer 1.F whose outer guard only fires when the command text contains the literal `/tmp/claude-bulk-commit-sentinel-*.json` path.

**Root Cause**: Layer 1.F outer guard trigger regex matches only `/tmp/claude-bulk-*` path patterns. Absolute-path script invocations produce the `/tmp/` path only as script-internal output (not visible in the command text). The guard was designed assuming all sentinel writes would reference the `/tmp/` path literally in the Bash command.

**Success Criteria**: All 8 ACs pass with pytest evidence (AC1–AC7b).

## Root Cause Analysis

**Symptom**: An agent invoking `python3 /root/.claude/scripts/write-bulk-commit-sentinel.py` could create a valid sentinel without triggering Layer 1.F, enabling use of the `auto-bulk:` BLESSED_BRIDGE_RE bypass in the privilege guard.

**Root Cause**: `pretool-bash-safety.sh` Layer 1.F outer guard at line 671 checked only for the literal string `/tmp/claude-bulk-commit-sentinel-` in the command text. Absolute-path script invocations and `python3 -c` with dynamic path construction both evade this check.

**Root Cause Commit**: `1238f42` + `41063a1` (established the sentinel-check pattern; this cycle closes the forge-sentinel gap)

## Implementation

**Approach**: Extend Layer 1.F's outer OR condition to also trigger on commands containing the literal string `write-bulk-commit-sentinel.py` (regardless of path prefix). Add an M5 allowlist using an anchored no-arg bash `[[ =~ ]]` regex so the legitimate `/commit --bulk` Step 5 form (`source venv/bin/activate && python3 /root/.claude/scripts/write-bulk-commit-sentinel.py`) is still allowed, while any form with arguments (including `--sid FORGED`) is denied. Add two explicit DO NOT items to `changelog-analyst.md`.
