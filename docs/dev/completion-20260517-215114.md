## Root Cause Analysis

**Gap**: /push, /merge, and /pull dispatched wrapper scripts directly with no agentic analysis step. The /commit + changelog-analyst architecture (close-gate → commit grant → analyst subagent → push-gate token) had no analogue in the other three commands.

**Architecture**: /commit has 4 load-bearing components: (1) close-gate pre-authorization file check, (2) commit grant written before subagent dispatch, (3) changelog-analyst intelligent worker subagent, (4) push-gate token written post-commit to authorize /push. The new analyst pattern adds a similar structure to push/merge (pre-execution gate) and pull (post-execution advisory).

## Implementation

**Approach**: Created three dedicated analyst agent files and modified three command bodies. Each operation gets a purpose-built analyst matched to its specific risk surface.

**Files Created**:
- `agents/push-analyst.md` — pre-push analyst; reads commits-to-push range via `refs/remotes/<resolved_remote>/<branch>..HEAD` (not @{u}..HEAD to handle fork workflows); writes nonce-keyed grant to /tmp/agentic-commit/push-analyst/*/; 120s expiry; verdicts: approved/warn/blocked
- `agents/merge-analyst.md` — pre-merge analyst; reads diff stat + conflict markers; writes nonce-keyed grant; 60s expiry (merge state freshness); tip revalidation before merge.sh
- `agents/pull-analyst.md` — post-pull advisory analyst; NO Write tool (excluded at policy layer); dispatched only when HEAD changed AND no rebase state directory present (rebase_conflict exits early); reports integration summary

**Files Modified**:
- `commands/push.md` — added 5-step agentic dispatch protocol before push.sh; Chain A (push-gate) and Chain B (push-analyst grant) validated independently
- `commands/merge.md` — replaced bare Implementation section with 5-step protocol including tip revalidation and re-dispatch on grant expiry
- `commands/pull.md` — added rebase-state-directory discriminant for pull_exit_phase; dispatches pull-analyst only on success/stash_restoration_failed (never on rebase_conflict)
- `policies/tool-policy.v1.json` — added push-analyst, merge-analyst, pull-analyst role entries with path-restricted Write (pull-analyst excluded from Write entirely)
- `hooks/pretool-cp-checkin.py` — added three new roles to CP_AGENTS set
