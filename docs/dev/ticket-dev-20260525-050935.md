## Edge Cases & Risks

1. **Binding constraint regression**: `!docs/dev/` and `!docs/reference/` in dot-claude `.gitignore` must survive; adding `docs/codex/` must be done as a separate positive rule (not by expanding `docs/*` negation), since `docs/*` already gitignores everything and `!docs/dev/` and `!docs/reference/` are the exceptions. A separate `docs/codex/` rule is not needed in dot-claude — `docs/*` already ignores it. What IS needed is preventing `tests/generated/` and `docs/codex/` from leaking as untracked noise. The untracked `tests/generated/` and `docs/codex/` dirs appear because `tests/generated/` is NOT covered by `docs/*` (it's under `tests/`, not `docs/`). And `docs/codex/` IS covered by `docs/*` already — but if not (i.e. git status shows it as untracked), an explicit rule resolves it. Dev must verify this with `git check-ignore` before adding redundant rules.

2. **budget-management output/dashboards tracked HTML**: `output/dashboards/asset-tracker.html` and `spending-dashboard.html` are tracked (confirmed via `git ls-files`). The `output/` rule in budget-management `.gitignore` will prevent new files; tracked ones need `git rm --cached output/dashboards/asset-tracker.html output/dashboards/spending-dashboard.html`. Should Have S1.

3. **Harness hook scope**: The harness propagation hook (`session-gitignore-propagate.sh`) runs at SessionStart from the working directory. The hook must detect the git root of the CWD (not always `/root/.claude`). If the CWD is not a git repo or has no `.claude/`, it exits 0 silently.

4. **`*.png` in dot-claude**: The existing `.png` files are under `worktrees/overnight-20260501-d0ec784d/screenshots/` which is already gitignored by the `worktrees/` rule (line 80). Adding `*.png` is idempotent for those files but prevents future PNG accumulation in other locations.

5. **`/root/.claude.json` gitignore**: `/root/.gitignore` already has `.claude.json` gitignored, so changes to this file are not tracked. Dev must edit the live file; no git operations needed on this file.

6. **orchestra `docs/research/case-*/` pattern**: Uses glob `case-*/` which matches `case-39742/` and future `case-NNNNN/` dirs. Does NOT match `case.md` or `case-notes.md`; INDEX.md and README.md at `docs/research/` level remain tracked.

- **M10** — `hooks/session-gitignore-propagate.sh` skeleton:
  ```bash
  #!/usr/bin/env bash
  # SessionStart hook: append missing standard harness gitignore rules to project repo
  # Runs idempotently — only appends rules that are absent.
  set -euo pipefail

  HARNESS_RULES=(
    "tests/generated/"
    ".claude/dev-registry/dev-*/"
    "docs/codex/"
    ".playwright-mcp/"
  )

  # Find git root of CWD
  GIT_ROOT=$(git rev-parse --show-toplevel 2>/dev/null) || exit 0
  # Only run for repos with .claude/ directory
  [[ -d "$GIT_ROOT/.claude" ]] || exit 0
  GITIGNORE="$GIT_ROOT/.gitignore"
  [[ -f "$GITIGNORE" ]] || exit 0

  for rule in "${HARNESS_RULES[@]}"; do
    if ! grep -qF "$rule" "$GITIGNORE"; then
      printf '# harness-propagated by session-gitignore-propagate.sh\n%s\n' "$rule" >> "$GITIGNORE"
    fi
  done
  exit 0
  ```
  Make executable: `chmod 755 hooks/session-gitignore-propagate.sh`
