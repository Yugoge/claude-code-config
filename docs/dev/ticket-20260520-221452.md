## Edge Cases & Risks

- **Risk — `git rm --cached` blast radius**: Removing `agent-scores.json` from tracking deletes it from any fresh checkout. The runtime-create-if-missing path in `score-update.sh:53-55` handles this (creates `{"global":{"agents":{}},"projects":{}}` skeleton on first invocation), but score-inject.sh:35-41 also handles the missing-file case (emits a neutral header). - **Risk — `git rm --cached .claude/specs/spec-20260518-225715`**: This removes 47 tracked cp-state and lock files. These are runtime telemetry — they will be re-created by the next spec invocation. Per-cycle scoping (lock files are unique per cycle) means there is no cross-cycle dependency.
- **Risk — manifest contract change**: Changing qa.md Phase 5 + commands/dev.md to read per-task manifest is a non-backward-compatible read change. Cycle 1 produced no `tests/generated/<task_id>/manifest.json` files (because complexity_tier was COMPLEX but test-writer was never actually dispatched — that's the consumability bug). So there is no data to migrate. The per-task path is the spec-aligned shape from Cycle 1 design; this fix is the first cycle where it actually matters.
- **Risk — venv subshell semantics**: The heredoc form `python3 - "$arg" <<'PYEOF'` runs python3 as a child of the current shell. `source venv/bin/activate` in the parent shell sets VIRTUAL_ENV which is inherited by python3 via env. Subshell wrap `( source ... && python3 - "$arg" <<'PYEOF' ... PYEOF )` is the most explicit form; the heredoc body remains inside the subshell so the activation propagates. Dev must verify by running the modified script and confirming `python3 -c 'import sys; print(sys.prefix)'` reports the venv path (not /usr).
- **Risk — Sub-step 12.0 rename impacts spec_section_updates logic**: Sub-step 12.0 currently triggers `apply_spec_section_updates` workflow per `commands/dev.md:822-843`. Renaming the label MUST preserve the procedural semantics; only the heading text changes, not the conditional logic or the orchestrator-side trigger. Dev MUST read the full block (lines 822-843) before edit to confirm no hook/command depends on the literal label "Sub-step 12.0".
- **Risk — Standard 6 exemption scope creep**: Codex #10 noted the exemption could be over-broad. The AC requires the exemption to NAME the exempted artifact classes (rank labels, AskUserQuestion choices) and explicitly RE-AFFIRM that code comments + stderr diagnostics remain English-only. This preserves Standard 6's protective intent for non-user-facing strings.
