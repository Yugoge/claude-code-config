### Codex findings — per-finding adjudication (Rule 3)

**BLOCKER #2 — repair-venv.sh overwrites real non-symlink files. → `in_scope` → BLOCKS close.**
- Codex evidence: `scripts/repair-venv.sh:143-152`. QA static confirmation: the `else` branch (line 143) is reached by ANY path that resolves to something, including a real regular file; if `"${TARGET}" --version` fails (line 145) → `needs_repair=1`; line 150 `rm -f "${TARGET}"` deletes the real file and symlinks over it, with no `! -L` guard.
- Why it blocks: a `rm -f` of a non-symlink, non-broken file is a silent destructive data-loss path. QA's anti-fraud disciplines ("silent-corrupt fail mode", "a false PASS is worse than a false FAIL") forbid passing it. The user's literal threat model is "broken symlinks" and the live venv has only symlinks (verified: python→python3, python3→/usr/bin/python3.12, python3.12→python3) — but a destructive edge in a shipped repair tool is not acceptable to wave through.
- **PROPOSED_FIX (codex, accepted)**: guard `[[ -e "$TARGET" && ! -L "$TARGET" ]]` → abort with "refusing to overwrite non-symlink: $TARGET" + exit 1 instead of `rm -f`. Mutate only absent paths and symlinks.

**MAJOR #3 — destructive venv-repair path has no automated test. → `in_scope` → fix with #2.**
- `test_AC2_*.py` exercises only the idempotent path; the destructive branch carrying the #2 bug is untested — which is precisely why the overwrite defect shipped.
- **PROPOSED_FIX (codex, accepted)**: add `--venv` throwaway-venv tests — broken symlink → repaired; real non-symlink file → preserved + non-zero exit.

**BLOCKER #1 — exit-2 binding documented/asserted, not end-to-end tested. → `in_scope_minor` → does NOT block alone.**
