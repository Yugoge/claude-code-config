## Edge Cases & Risks

- **Risk 1**: Rounding positive deltas during 1/3 scaling may produce sub-optimal granularity. Mitigation: use the specific integer values in M1 (pre-computed with careful rounding).
- **Risk 2**: `<options>` XML in /close output triggers the `pretool-orchestrator-prompt-purity.py` warn-only hook when dispatching via Agent. Mitigation: the `<options>` block is in /close's OWN text output (not a subagent dispatch prompt), so the hook does not apply -- /close is a command, not a dispatch.
- **Risk 3**: The session summary content is LLM-generated and may be inaccurate. Mitigation: the summary is advisory context for the user, not a contractual artifact.
- **Risk 4**: The staging whitelist in M5/M6 may reject legitimate files that the dev-report omits. Mitigation: the +30 artifact overhead buffer, plus cycle artifacts matching the anchored patterns enumerated in M5 (e.g. `ticket-<TASK_ID>.md`, `context-<TASK_ID>.json`) are explicitly included in the whitelist.
- **Risk 5**: If `baseline_head_sha` is absent (older dev-reports), the provenance sanity check cannot function. Mitigation: when dev-report exists but `baseline_head_sha` is absent, the whitelist and foreign-session exclusion are still enforced; only the provenance sanity check is skipped. When no dev-report exists at all, behavior falls back to current stage-everything with a WARNING.
