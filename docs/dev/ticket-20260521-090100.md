## Goal

Establish `docs/codex/<task_id>/<role>.txt` as the canonical Codex output path. Update `tool-policy.v1.json` to grant write access to `*/docs/codex/` for the 11 roles that invoke Codex consultation (excluding `dev`, which already has wildcard write access). ## Context

Post-mortem of cycle 20260519-211515 surfaced that subagents invoking the `codex` Skill are blocked when writing output because `bash_write_targets.py::_resolve_path()` (lines 152-170) expands ONLY `$HOME`, `~`, `$CLAUDE_PROJECT_DIR` — it cannot expand arbitrary shell variables like `$CODEX_OUT`. The hook sees the literal string `$CODEX_OUT` as the tee target, which matches no allowed prefix. The fix requires: (a) changing the tee target to a hook-resolvable literal form (Steps 3/4 branched directly, NOT via `$CODEX_OUT` indirection), AND (b) adding `*/docs/codex/` as an allowed prefix for consultation roles.

**Note on resolvability**: Codex-verified (session 019e4f2d) that `tee "$CLAUDE_PROJECT_DIR/docs/codex/$CODEX_OUT_TASK_ID/$CODEX_OUT_ROLE.txt"` IS correctly resolved by the policy checker. `_resolve_path()` expands `$CLAUDE_PROJECT_DIR` to the actual project dir; `policy_registry._glob_match()` auto-appends `*` for `*/docs/codex/` prefix; `fnmatch.fnmatchcase` matches the resolved path even with unexpanded `$CODEX_OUT_TASK_ID/$CODEX_OUT_ROLE.txt` suffix. Policy-level permission check passes correctly.

**Prior ticket reconciliation**: `docs/dev/ticket-20260521-085827-fa9f07.md` (created 2026-05-21T08:58:27Z) analyzed the same ITEM #1 with compatible conclusions: SKILL.md required, path-safety validation required, dev excluded per CR-1.7 (wildcard `"*"`). ## Edge Cases & Risks

1. **Hook variable-expansion**: `_resolve_path()` expands ONLY `$HOME`, `~`, `$CLAUDE_PROJECT_DIR`. The tee target MUST NOT be a standalone variable like `$CODEX_OUT`. Use the pre-validated env var values directly in the tee argument as `"$CLAUDE_PROJECT_DIR/docs/codex/$CODEX_OUT_TASK_ID/$CODEX_OUT_ROLE.txt"`.
2. **Path-safety traversal**: `CODEX_OUT_TASK_ID` or `CODEX_OUT_ROLE` containing `/`, `..`, or special chars must trigger fallback, not traverse `docs/codex/`. The `^[A-Za-z0-9._-]+$` guard is mandatory.
3. **Worktree paths**: `*/docs/codex/` matches under any worktree root, consistent with other entries.
4. **Legacy callers**: subagents not exporting `CODEX_OUT_TASK_ID` / `CODEX_OUT_ROLE` fall back gracefully.
5. **SKILL.md is outside dot-claude repo**: at `/root/.agents/skills/codex/`. Dev's wildcard `"*"` covers writes there.
6. **policy_version**: starting from confirmed baseline of 3. If a concurrent policy edit has occurred, dev must handle manually.

## Technical Hints

- **Affected files (3)**:
  - `policies/tool-policy.v1.json` — 11 role entries + `policy_version` 3→4 + `_note` amendment
  - `commands/codex.md` — Step 2 path-resolution block; all three tee sites in Steps 3/4

- **Path-resolution bash block (suggested shape for Step 2 in both files)**:
  ```bash
  mkdir -p /var/tmp/codex-outputs
  _safe_re='^[A-Za-z0-9._-]+$'
  if [ -n "${CODEX_OUT_TASK_ID:-}" ] && [ -n "${CODEX_OUT_ROLE:-}" ] \
      && [[ "$CODEX_OUT_TASK_ID" =~ $_safe_re ]] \
      && [[ "$CODEX_OUT_ROLE" =~ $_safe_re ]]; then
    _base="${CLAUDE_PROJECT_DIR:-$PWD}/docs/codex/$CODEX_OUT_TASK_ID"
    mkdir -p "$_base"
    CODEX_OUT="$_base/$CODEX_OUT_ROLE.txt"
  else
    if [ -n "${CODEX_OUT_TASK_ID:-}${CODEX_OUT_ROLE:-}" ]; then
      echo "[codex skill] warning: env vars present but unsafe; falling back to /var/tmp/codex-outputs/" >&2
    fi
    CODEX_OUT="/var/tmp/codex-outputs/codex-output-$$-$(date +%s).txt"
  fi
  echo "$CODEX_OUT"
  ```

- **Tee restructuring strategy**: since `tee "$CODEX_OUT"` is forbidden (hook sees literal `$CODEX_OUT`), dev must restructure steps 3/4 so each tee command directly embeds the resolvable path form. Recommended: use `"$CLAUDE_PROJECT_DIR/docs/codex/$CODEX_OUT_TASK_ID/$CODEX_OUT_ROLE.txt"` (canonical branch) or the literal fallback path. When `CLAUDE_PROJECT_DIR` is set, `_resolve_path()` will expand this correctly. When absent, the `${CLAUDE_PROJECT_DIR:-$PWD}` variant works at the policy-check level.
