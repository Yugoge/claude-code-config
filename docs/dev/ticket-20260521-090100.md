## Goal

Establish `docs/codex/<task_id>/<role>.txt` as the canonical Codex output path. Update `tool-policy.v1.json` to grant write access to `*/docs/codex/` for the 11 roles that invoke Codex consultation (excluding `dev`, which already has wildcard write access). Update both `commands/codex.md` and `/root/.agents/skills/codex/SKILL.md` so the default output target uses that canonical path, with path-safety validation and a fallback to `/var/tmp/codex-outputs/`.

## Context

Post-mortem of cycle 20260519-211515 surfaced that subagents invoking the `codex` Skill are blocked when writing output because `bash_write_targets.py::_resolve_path()` (lines 152-170) expands ONLY `$HOME`, `~`, `$CLAUDE_PROJECT_DIR` — it cannot expand arbitrary shell variables like `$CODEX_OUT`. The hook sees the literal string `$CODEX_OUT` as the tee target, which matches no allowed prefix. The fix requires: (a) changing the tee target to a hook-resolvable literal form, AND (b) adding `*/docs/codex/` as an allowed prefix for consultation roles.

Prior ticket `docs/dev/ticket-20260521-085827-fa9f07.md` (created 2026-05-21T08:58:27Z, 27 minutes before this ticket) analyzed the same problem as ITEM #1 and reached compatible conclusions: SKILL.md also required, path-safety validation required, dev excluded per CR-1.7 (wildcard `"*"`). This spec supersedes it on one point: the prior ticket's role count of 9 missed `spec` and `user` agents (both invoke `Skill(skill="codex")` per `agents/spec.md:693` and `agents/user.md:545`), making the correct count 11 roles.

## Edge Cases & Risks

1. **Hook variable-expansion ambiguity (Codex finding #2)**: the bash write-target parser in `lib/bash_write_targets.py` does not expand arbitrary shell variables. If `commands/codex.md` Step 2 sets `CODEX_OUT=docs/codex/...` and Step 4 uses `tee "$CODEX_OUT"`, the hook sees literal `$CODEX_OUT` and blocks. **Mitigation**: Step 4 in `commands/codex.md` must inline the path directly (not via variable), OR Step 2 must print the path and Step 4 must use command substitution in a single literal-resolved form. The spec requires dev to verify the hook parser sees the resolved path.
2. **test-writer has Skill but is not a codex-consultation role**: do not add `*/docs/codex/` to test-writer. The exclusion criterion is "codex consultation per agents/*.md" not "has Skill tool."
3. **timeline and transportation have Skill in tool-policy but denied_tools includes Write/Edit**: adding `*/docs/codex/` would be vacuous for them (Write is denied). Excluding them is correct.
4. **policy_version increment**: tool-policy is at version 3. Dev must increment to 4 and add a `_note` sentence explaining the change.
5. **Directory creation**: `docs/codex/<task_id>/` does not exist at repo level and need not be created. `commands/codex.md` Step 2 must include `mkdir -p docs/codex/<task_id>` before the tee command.
6. **Worktree paths**: the `*/docs/codex/` prefix (with leading `*/`) matches the path under any worktree root, consistent with other entries.
