## Git Protection Architecture

`.claude/` enforces an always-on git guard that refuses agent-issued
`git commit/merge/push/reset --hard` except via three sanctioned paths:
blessed-bridge regex (legacy + overnight), env-var (`/merge`), or
env-var + single-use grant manifest (`/commit`, `/push`).

Wrappers (`hooks/commit.sh`, `hooks/push.sh`) own grant emission and
unlinking; `commit.sh` has a closed-task mode (`/commit <task-id>`) and
a bridge mode (`/commit --auto-bulk-bridge <branch>`) for `/dev-overnight`.

`/close` debates enforce four Workflow Integrity bullets in addition to
the usual AC-met / root-cause / regression dimensions.

Full architecture, 13 sanctioned/attack scenarios, mermaid diagrams, and
critical invariants: **`/root/docs/git-protection-architecture.md`**.
